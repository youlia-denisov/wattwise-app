Overview

What the app does

WattWise turns a raw CSV export from the Israel Electric Company into a rich, multi-tab analytical report. It runs a seven-step pipeline automatically — from parsing and cleaning the data all the way through machine learning, weather integration, and personalised discount recommendations. Every result stays private: each browser session gets its own isolated workspace.

📊

Usage Patterns

Daily totals, weekday heatmaps, and hourly breakdowns reveal when and how much electricity you consume.

🤖

Behaviour Clustering

K-Means groups your readings into labelled consumption profiles — morning heavy, night owl, weekend peak, and more.

🔍

Outlier Detection

Four methods run in parallel — 3σ, IQR, DBSCAN, and Isolation Forest — with an auto-selected recommendation.

🌤️

Weather Correlation

Historical temperature data from Open-Meteo is merged with your consumption to show climate-driven usage spikes.

💰

Discount Analysis

Live electricity discount offers are scraped weekly and matched against your actual data to show real NIS savings per plan.

🧮

Savings Calculator

Enter custom monthly usage or use your real data to interactively compare which discount plan saves you the most.

Demo mode is available without uploading a file. Several features — clustering, outlier comparison, weather, and personalised discount savings — require a real IEC export to unlock full results.

Under the hood

How the pipeline works

When you click "Run Full Pipeline", seven steps execute in sequence. Each step writes its output to your private session folder, so the dashboard can reload results instantly on any tab you open.

1
Data

Load & Clean

Auto-detects the header row in any IEC CSV export, isolates date/time/kWh columns, coerces types, interpolates short gaps, and adds time-feature columns (hour, weekday, month).

2
Machine Learning

K-Means Clustering

Encodes cyclical time features (hour, weekday), scales the feature matrix, and fits K-Means. Clusters are ranked and assigned human-readable consumption-profile labels.

3
Statistics

Aggregation & Outlier Detection

Computes hourly stats, daily stats, and daily totals. Runs 3σ and IQR outlier detection in parallel and saves flagged readings for the trends tab.

4
Visualisation

Chart Generation

Saves all Plotly charts as self-contained HTML files and heatmap comparisons as PNGs — so the dashboard loads pre-rendered visuals instantly without recomputing.

5
External API

Weather Integration

Fetches historical hourly temperature data from Open-Meteo for your city, merges it with consumption, and generates dual-axis correlation charts. Optional step.

6
Analysis

Discount Scenario Modelling

Matches each live electricity plan's discount hours and days against your actual readings, computes eligible kWh, and estimates annual NIS savings at the configured tariff.

7
Report

Summary Report

Assembles a Markdown report from all pipeline outputs — summary stats, top outliers, weather insights, and the best discount recommendation — readable in the Report tab.

Architecture

How it's structured

The codebase is split into four layers. Tabs receive pre-loaded DataFrames as arguments — they never read files directly — which keeps the UI fast and the data logic testable independently.

UI Layer

Streamlit tabs — overview, hourly, clustering, outliers, weather, discounts, calculator, report.

Orchestration

Pipeline runner, cached data loaders, and weekly offer refresh logic with scraper fallback.

Processing (src/)

Loader, preprocessing, aggregation, clustering, features, outlier pipeline, discount analysis, weather, reporting.

Data Layer

Per-user temp directories, pipeline output CSVs, pre-rendered HTML charts, and Open-Meteo API.

Multi-user isolation is handled by tempfile.mkdtemp() at session start. All pipeline outputs go into that directory, and @st.cache_data uses the directory path as part of the cache key — so two users never share cached results.
